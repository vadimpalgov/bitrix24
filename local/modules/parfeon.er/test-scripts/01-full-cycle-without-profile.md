# Сценарий 1 — Полный цикл согласования без профиля

Базовый happy path: заявка проходит все три фазы согласования через автоматическую иерархию.  
Профиль согласования у заявителя **не назначен**.

## Предусловия

- [ ] Настройки модуля заполнены: `ER_SMART_PROCESS_ID=1046`, `AP_SMART_PROCESS_ID=1050`
- [ ] Стадии ER и AP настроены в options.php
- [ ] `LA_EXCLUDE_DIRECTOR = N` (директор включён в цепочку)
- [ ] `LA_ADD_ALL_MANAGERS = Y` (вся цепочка иерархии)
- [ ] У Иванова (`ivanov`, ID 5) поле `UF_APPROVAL_PROFILE` **пустое**
- [ ] Иванов находится в отделе с руководителем Дмитриевым по иерархии HumanResources
- [ ] Анна Сидорова и Екатерина Круглова находятся в подразделении с именем `HR`

## Шаги и ожидаемые результаты

### Шаг 1 — Создание заявки

**Действие:** войти под `ivanov` (ID 5), создать новый элемент смарт-процесса «Заявки сотрудников».  
Заполнить:
- Тип заявки: «Заявка на отпуск»
- Дата начала: через 14+ дней
- Дата окончания: через 28+ дней
- Руководители проекта (`UF_CRM_10_PROJECT_MANAGERS`): Олег Орлов (`orlov`, ID 19)
- Описание: произвольный текст

**Ожидаемый результат:**
- [ ] Элемент создан, заголовок автоматически: `«Заявка на отпуск от Иван Иванов»`
- [ ] Поле `UF_CRM_10_HEAD_OF_DEPARTMENT` заполнено автоматически: [4, 34] (Дмитриев, Генералов)
- [ ] Поле `UF_CRM_10_HR` заполнено автоматически: [21, 22] (Сидорова, Круглова)
- [ ] AP-элементы **не созданы** (заявка ещё не в работе)
